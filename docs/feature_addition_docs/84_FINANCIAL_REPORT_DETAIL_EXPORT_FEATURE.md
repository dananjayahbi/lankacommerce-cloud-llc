# Financial Report Detail & Export Feature Specification

## Backend API Requirements

### Endpoints:

**GET /api/accounting/reports/{id}/**
- Retrieve report details
- Response: `{ id, type, period_start, period_end, file_format, file_size, file_url, generated_at, generated_by, status, parameters, comparison_period (nullable) }`

**GET /api/accounting/reports/{id}/download/**
- Download report file
- Query params: format (optional, for export to different format)
- Response: file download

**GET /api/accounting/reports/{id}/export-as/**
- Export report to different format
- Query params: format (pdf/excel/csv)
- Response: file download (may be async for large files)

**POST /api/accounting/reports/{id}/email/**
- Email report
- Request body: `{ recipients: [email], subject, message (optional), file_format }`
- Response: `{ success: true, message: "Report emailed" }`

**POST /api/accounting/reports/{id}/verify/**
- Mark report as verified
- Request body: `{ comments (optional) }`
- Response: `{ id, status, verified_at, verified_by }`

**POST /api/accounting/reports/{id}/unverify/**
- Revert verification
- Response: `{ id, status }`

**POST /api/accounting/reports/{id}/archive/**
- Archive report
- Response: `{ id, archived_at }`

**POST /api/accounting/reports/{id}/unarchive/**
- Unarchive report
- Response: `{ id, archived_at: null }`

**DELETE /api/accounting/reports/{id}/**
- Delete report
- Response: `{ success: true, message: "Report deleted" }`

**GET /api/accounting/reports/{id}/drill-down/**
- Get underlying data (if applicable)
- Query params: account_id, transaction_type
- Response: [transactions]

**POST /api/accounting/reports/{id}/comments/**
- Add comment to report
- Request body: `{ comment_text }`
- Response: `{ id, user, timestamp, comment }`

## Validation & Edge Cases

- Report must exist and be accessible by user
- File must exist in storage
- Export may require async job for large files
- Email delivery requires valid recipients
- Verification requires authorization
- Archived reports are read-only
- Multi-format export may regenerate file
- Large reports may need pagination in viewer
- Account drilling only for summary rows
- Comparison only if comparison period set
- Comments require proper permissions
