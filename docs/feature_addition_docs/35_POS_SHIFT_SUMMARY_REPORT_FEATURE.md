# POS Shift Summary Report Feature Specification

## Executive Summary

Shift summary report providing comprehensive audit trail of cashier performance with transaction details, payment breakdown, variances, and actions for shift reconciliation and compliance. This feature enables detailed analysis of shift operations, facilitates dispute resolution, and maintains regulatory compliance through complete documentation.

---

## Current State

Basic report display is partially implemented:
- Transaction summary calculation is working
- Payment method breakdown is incomplete
- Variance display is functional
- PDF generation is partially implemented
- Email reporting is not implemented
- Report caching/optimization not in place
- Report sharing and export features incomplete

---

## Detailed Requirements

### Frontend Features

#### 1. Report Header Section (Read-Only)

- **Report Title**
  - Display "Shift Summary Report"
  - Large, professional heading
  - Clear visual hierarchy

- **Report Generation Metadata**
  - Report generation date/time (timestamp)
  - Report generated by (user name or system)
  - Report version/reference number
  - Unique report ID for tracking

- **Shift Date and Time Range**
  - Shift date in locale format (e.g., "May 31, 2026")
  - Opening time (e.g., "9:00 AM")
  - Closing time (e.g., "5:30 PM")
  - Duration calculation (total hours/minutes)

#### 2. Cashier Information Section

- **Cashier Name**
  - Display assigned cashier
  - Format: "First Last"

- **Employee ID**
  - Unique employee identifier
  - Link to employee profile (optional)

- **Terminal ID / Location**
  - Terminal name/number
  - Store location if applicable
  - Terminal timezone

#### 3. Transaction Summary Section

- **Opening Cash Amount**
  - Amount recorded at shift open
  - Currency formatted

- **Opening Time**
  - Timestamp of shift open
  - Includes date

- **Closing Cash Amount**
  - Amount counted at shift close
  - Currency formatted

- **Closing Time**
  - Timestamp of shift close
  - Includes date

- **Total Work Hours** (Auto-Calculated)
  - Duration calculation (closing time - opening time)
  - Format: "X hours Y minutes"
  - Adjust for shift breaks if applicable

#### 4. Sales Breakdown Section

- **Total Transactions Count**
  - Number of all transactions in shift
  - Exclude non-transaction activities

- **Total Sales Amount**
  - Sum of all sales transactions
  - Format with currency
  - Exclude refunds from this total

- **Average Transaction Value** (Auto-Calculated)
  - Formula: Total sales / Transaction count
  - Format with currency
  - Display to 2 decimal places

- **Payment Method Breakdown Table**
  - Columns:
    - Payment method name (Cash, Credit Card, Debit Card, Check, etc.)
    - Transaction count (number of transactions)
    - Total amount (sum of transactions for method)
    - Percentage of total (e.g., "45.3%")
  - Support multiple payment types
  - Sort by amount descending
  - Include subtotal row if applicable
  - Calculate percentages accurately

#### 5. Deductions Breakdown Section

- **Total Discounts Given**
  - Sum of all discount transactions
  - Format with currency
  - Include count of discount transactions

- **Total Refunds Processed**
  - Sum of all refund transactions
  - Format with currency
  - Include count of refund transactions

- **Total Taxes Collected**
  - Sum of all taxes on transactions
  - Format with currency
  - May be calculated or recorded

- **Net Sales** (Auto-Calculated)
  - Formula: Total sales + Discounts - Refunds
  - Format with currency
  - Prominently displayed
  - Indicates performance metric

#### 6. Cash Reconciliation Section

- **Opening Balance**
  - Displayed from opening cash

- **Plus: Cash Sales**
  - Total cash transactions

- **Less: Refunds**
  - Total refunds (negative amount)

- **Expected Closing Balance** (Auto-Calculated)
  - Formula: Opening + Cash sales - Refunds
  - Format with currency

- **Actual Closing Balance**
  - Amount counted by cashier

- **Variance Amount**
  - Difference: Actual - Expected
  - Format with currency
  - Show as +/- sign

- **Variance Status**
  - "Balanced" - if zero variance
  - "Variance Exists" - if non-zero
  - Display prominently
  - Color coding (green for balanced, red for variance)

#### 7. Variance Details Section (If Applicable)

- **Display**: Only if variance exists

- **Variance Amount**
  - Repeat variance from reconciliation

- **Variance Reason**
  - Show selected reason from shift close
  - Link to reason documentation

- **Variance Investigation Notes**
  - Display free-text notes entered at close
  - Format as readable text block

- **Resolved By**
  - User who investigated variance
  - Timestamp of investigation

- **Variance Status**
  - "Resolved", "Pending", or "Escalated"
  - Indicates investigation state

#### 8. Action Buttons

- **Print Button**
  - Generates printer-ready format
  - Removes interactive elements
  - Optimized for paper output
  - Preserves all data in readable format

- **Email Report Button**
  - Opens email dialog
  - Pre-populated recipient based on user role
  - Options to add additional recipients
  - Allows custom message
  - Sends as attachment or link

- **Download PDF Button**
  - Generates PDF file
  - Professional formatting
  - Preserves all report data
  - Download to device

- **Share Report Button**
  - Generates shareable link
  - Time-limited access (configurable)
  - View-only permissions
  - Tracks access attempts

- **Export to CSV Button**
  - Exports all report data
  - Includes detailed transaction list (if included)
  - Supports further analysis
  - Maintains data integrity

### Backend API Requirements

#### GET /api/pos/shifts/{shift_id}/report/
Generate shift summary report
- **Response** (200 OK):
  ```
  {
    "report_id": "uuid",
    "shift_id": "uuid",
    "generated_at": "ISO 8601 datetime",
    "shift": {
      "opening_cash": "decimal",
      "opening_time": "ISO 8601 datetime",
      "closing_cash": "decimal",
      "closing_time": "ISO 8601 datetime",
      "cashier_name": "string",
      "terminal_id": "string"
    },
    "summary": {
      "total_transactions": "integer",
      "total_sales": "decimal",
      "average_transaction_value": "decimal",
      "total_discounts": "decimal",
      "total_refunds": "decimal",
      "total_taxes": "decimal",
      "net_sales": "decimal"
    },
    "payment_methods": [
      {
        "method": "string",
        "count": "integer",
        "total": "decimal",
        "percentage": "decimal"
      }
    ],
    "reconciliation": {
      "opening_balance": "decimal",
      "cash_sales": "decimal",
      "refunds": "decimal",
      "expected_closing": "decimal",
      "actual_closing": "decimal",
      "variance": "decimal",
      "variance_status": "string"
    },
    "variance": {
      "amount": "decimal",
      "reason": "string",
      "notes": "string",
      "resolved_by": "uuid",
      "resolved_at": "ISO 8601 datetime"
    }
  }
  ```

#### GET /api/pos/shifts/{shift_id}/transactions/
Get all transactions in shift
- **Query Parameters**:
  - offset (optional, pagination)
  - limit (optional, results per page)
  - transaction_type (optional, filter)
- **Response** (200 OK):
  ```
  {
    "transactions": [
      {
        "id": "uuid",
        "timestamp": "ISO 8601 datetime",
        "type": "sale|refund|discount",
        "amount": "decimal",
        "payment_method": "string",
        "item_count": "integer",
        "items": [
          {
            "name": "string",
            "quantity": "integer",
            "unit_price": "decimal",
            "total": "decimal"
          }
        ],
        "reference": "string"
      }
    ],
    "total_count": "integer",
    "offset": "integer",
    "limit": "integer"
  }
  ```

#### GET /api/pos/shifts/{shift_id}/variance/
Get variance details
- **Response** (200 OK):
  ```
  {
    "variance_amount": "decimal",
    "reason": "string",
    "notes": "string",
    "resolved_by": "uuid",
    "resolved_at": "ISO 8601 datetime",
    "investigation_log": [
      {
        "timestamp": "ISO 8601 datetime",
        "user": "string",
        "action": "string",
        "notes": "string"
      }
    ]
  }
  ```

#### POST /api/pos/shifts/{shift_id}/report/email/
Send report via email
- **Request Body**:
  ```
  {
    "recipients": ["email@example.com"],
    "message": "string (optional)",
    "include_transactions": "boolean (optional)",
    "format": "html|pdf|csv"
  }
  ```
- **Response** (200 OK):
  ```
  {
    "email_id": "uuid",
    "status": "sent",
    "recipients": ["email@example.com"],
    "sent_at": "ISO 8601 datetime"
  }
  ```

#### POST /api/pos/shifts/{shift_id}/report/pdf/
Generate PDF report
- **Request Body**:
  ```
  {
    "include_transactions": "boolean (optional)",
    "include_variance": "boolean (optional)"
  }
  ```
- **Response** (200 OK - Binary PDF Data):
  - Content-Type: application/pdf
  - Content-Disposition: attachment; filename="shift_report_[date].pdf"

#### POST /api/pos/shifts/{shift_id}/report/share/
Generate shareable link
- **Request Body**:
  ```
  {
    "expiration_hours": "integer (optional, default: 24)"
  }
  ```
- **Response** (200 OK):
  ```
  {
    "share_url": "string",
    "expires_at": "ISO 8601 datetime",
    "share_token": "string"
  }
  ```

### Database Requirements

#### Report Data Model (Query from Multiple Sources)

The report aggregates data from:
- **POSShift**: Base shift information
- **POSTransaction**: All transactions in shift
- **POSVarianceLog**: Variance details if applicable
- **POSPaymentMethod**: Payment method definitions

#### Report Caching Model
```
POSShiftReport:
- id (UUID, Primary Key)
- shift_id (UUID, Foreign Key)
- report_data (JSON) - Cached report content
- generated_at (DateTime)
- generated_by (UUID)
- expires_at (DateTime, for cache TTL)
- created_at (DateTime)

Indexes:
- (shift_id) - Lookup reports for shift
- (generated_at) - For cache management
- (expires_at) - For cleanup queries
```

#### Report Access Audit
```
POSReportAccess:
- id (UUID, Primary Key)
- report_id (UUID, Foreign Key)
- accessed_by (UUID, Foreign Key)
- access_time (DateTime)
- access_method (print|email|download|view)
- ip_address (String, nullable)
- created_at (DateTime)

Indexes:
- (report_id) - Lookup accesses for report
- (accessed_by) - Track user actions
- (access_time) - Temporal analysis
```

## Validation & Edge Cases

### Report Generation Validation

1. **Empty Shift Handling**
   - Shift with no transactions
   - All calculations should be zero or N/A
   - Display gracefully without errors
   - Reconciliation shows no activity

2. **Shift with Only Cash Transactions**
   - Payment method breakdown shows only cash
   - Card transaction count = 0
   - Net sales = opening + cash - refunds
   - Other methods not in breakdown

3. **Shift with Only Card Transactions**
   - Payment method breakdown shows only card
   - Cash reconciliation may show variance
   - No cash sales component
   - Net sales reflects card transactions

4. **Mixed Payment Methods**
   - Correctly aggregate all payment types
   - Percentage calculations sum to 100%
   - Payment method ordering (largest first)
   - All methods included in breakdown

5. **Large Transaction Count**
   - Performance impact with 5000+ transactions
   - Pagination for transaction list
   - Summary calculations remain fast
   - Report generation <2 seconds

6. **Report Generation Timeout**
   - Handle gracefully if generation slow
   - Display partial report if available
   - Generate detailed version asynchronously
   - Queue PDF generation if needed

7. **Concurrent Report Generation**
   - Prevent duplicate processing
   - Cache results to avoid re-generation
   - Return cached report if exists
   - Update cache only after full generation

8. **Report Data Accuracy**
   - Ensure calculations match live data
   - Verify totals against source transactions
   - Validate variance calculations
   - Cross-check payment method sums

9. **Timezone Display**
   - Ensure consistent timezone in reports
   - Display times in terminal timezone
   - Include timezone in header
   - Convert if viewing from different timezone

10. **Multi-Language Report Generation**
    - Support multiple languages if applicable
    - Translate payment method names
    - Maintain number formatting
    - Handle right-to-left languages

### Edge Cases

- **Very Long Shift** (20+ hours)
  - Still calculate correctly
  - Display duration properly
  - No performance issues

- **Late Night Shift** (11 PM - 7 AM)
  - Cross-day transactions
  - Timezone calculations correct
  - Date display accurate

- **Shift with No Closing**
  - Still generate report if shift closed
  - Show "Shift Open" in status if not closed
  - Allow report generation for unclosed shifts

- **Shift with Large Variance**
  - Display variance prominently
  - Investigation notes fully shown
  - Reconciliation remains clear

- **Shift with Refunds Only**
  - Handle negative net sales gracefully
  - Display refund count/total
  - Reconciliation shows refund impact

## Monitoring & Alerting

### Key Metrics

1. **Report Generation Latency**
   - Track P50, P95, P99 times
   - Alert if P95 >2.5s
   - Optimize slow queries

2. **PDF Generation Failures**
   - Track failure rate
   - Alert if >1% failure
   - Investigate causes

3. **Email Delivery Failures**
   - Track delivery failures
   - Alert if >5% failure
   - Check email service

4. **Report Access Frequency**
   - Track usage patterns
   - Identify popular reports
   - Monitor access spikes

5. **Cache Hit Rate**
   - Track cache effectiveness
   - Optimize TTL if needed
   - Monitor storage usage

6. **Slow Queries**
   - Alert on slow report queries
   - Monitor database performance
   - Optimize indexes if needed

7. **Storage Usage**
   - Track PDF storage growth
   - Set storage limits
   - Archive old reports

### Dashboard Metrics

- Real-time report generation success rate
- Average report generation time (P95)
- PDF generation failure rate
- Email delivery success rate
- Cache hit rate
- Total reports generated (daily/weekly)
- Active report viewers

### Alerting Rules

- **Critical**: Report generation failure (page on-call)
- **High**: Report generation >3s P95 (investigate)
- **High**: Email delivery failure >10% (check service)
- **Medium**: PDF generation failure >5% (investigate)
- **Medium**: Cache hit rate <50% (optimize)
- **Low**: Storage usage approaching limit (archive)
