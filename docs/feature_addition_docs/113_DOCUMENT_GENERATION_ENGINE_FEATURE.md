# Document 113: DOCUMENT GENERATION ENGINE FEATURE

## Executive Summary

Document Generation Engine provides intelligent document generation from transaction data with multi-format output (PDF, email HTML, print formats) enabling seamless document creation and distribution across the business.

## Current State Analysis

### EXISTING:
- Invoice PDF rendering service (InvoicePDFView in backend/apps/billing/views/)
- Report PDF export (using @react-pdf/renderer library)
- Generate and email invoice service (backend/apps/billing/services/invoice_service.py)
- PO service and utilities (backend/apps/crm/services/purchase_order_service.py, po_formatter.py)
- Email delivery service (backend/apps/billing/services/email_service.py)
- Document data models (Invoice, PO, Quote models)
- Serializers for document data (purchase_order_serializer, etc.)

## Backend API Requirements

- **POST /api/documents/invoices/{id}/generate/** - Generate invoice document
  - Request body: { template_id, format, options }
  - Response: { document_id, url or file }

- **POST /api/documents/purchase-orders/{id}/generate/** - Generate PO document
  - Request body: { template_id, format, options }
  - Response: document

- **POST /api/documents/quotes/{id}/generate/** - Generate quote document
  - Request body: { template_id, format, options }
  - Response: document

- **POST /api/documents/delivery-notes/{id}/generate/** - Generate delivery note
  - Request body: { template_id, format, options }
  - Response: document

- **POST /api/documents/batch-generate/** - Generate multiple documents
  - Request body: { document_ids, document_type, template_id, format }
  - Response: { job_id, status, documents: [...] }

- **GET /api/documents/{id}/preview/** - Get document preview
  - Query params: template_id, format
  - Response: { html_preview or pdf_url }

- **GET /api/documents/{id}/generated-history/** - Get generated documents history
  - Response: [{ id, format, generated_at, generated_by }]

- **GET /api/documents/variables-reference/** - Get available variables
  - Query params: document_type
  - Response: { variables: [...] }

## Database Requirements

- **GeneratedDocument model**: document_id, document_type, template_id, format, file_path, generated_at, generated_by, file_size, expiry_date
- **DocumentGenerationLog model**: document_id, template_id, format, status, generated_at, generated_by, error_message (if failed)
- **Indexes**: (document_id, generated_at DESC), (document_type, generated_at DESC)
