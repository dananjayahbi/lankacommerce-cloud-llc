## Backend API Requirements

- **GET /api/reports/saved-reports/** - Get list of saved reports
  - Query params: limit, offset, sort_by, filter_type, date_range
  - Response: [{ id, name, type, generated_at, generated_by, format, file_size, status }]

- **GET /api/reports/saved-reports/{id}/** - Get report details
  - Response: { id, name, type, generated_at, generated_by, format, file_size, status, file_url, parameters, metadata }

- **GET /api/reports/saved-reports/{id}/download/** - Download report file
  - Query params: format (optional, for conversion)
  - Response: file download

- **POST /api/reports/saved-reports/{id}/email/** - Email report
  - Request body: { recipients: [emails], subject, message (optional) }
  - Response: { success: true, message: "Report emailed" }

- **DELETE /api/reports/saved-reports/{id}/** - Delete report
  - Response: { success: true, message: "Report deleted" }

- **POST /api/reports/saved-reports/{id}/duplicate/** - Duplicate report (regenerate with same params)
  - Response: { id, status: "processing", message: "Report generation started" }

- **GET /api/reports/scheduled-reports/** - Get scheduled reports (if feature enabled)
  - Response: [{ id, report_type, schedule, next_run, last_run, email_recipients }]

- **POST /api/reports/scheduled-reports/** - Create scheduled report
  - Request body: { report_type, schedule (cron), email_recipients, parameters }
  - Response: { id, schedule, next_run }

- **PATCH /api/reports/scheduled-reports/{id}/** - Update scheduled report

- **DELETE /api/reports/scheduled-reports/{id}/** - Delete scheduled report

- **GET /api/reports/report-types/** - Get available report types
  - Response: [{ type, name, description, available_formats, default_parameters }]

- **POST /api/reports/generate/** - Generate custom report
  - Request body: { report_type, start_date, end_date, format, filters: {...}, parameters: {...} }
  - Response: { id, status: "processing", estimated_time_ms, file_url (when ready) }

- **GET /api/reports/{id}/status/** - Check report generation status
  - Response: { id, status, progress_percent, estimated_remaining_ms, error (if failed) }

- **GET /api/reports/cache-stats/** - Get cache statistics (admin only)
  - Response: { hit_rate, cache_size_mb, oldest_report_date, report_count_cached }

- **POST /api/reports/clear-cache/** - Clear report cache (admin only)
  - Response: { success: true, cleared_count, freed_space_mb }

## Validation & Edge Cases

- Report file must exist before download
- Email recipients must be valid
- Scheduled reports require valid cron expression
- Report type must be valid
- Filters must match available fields
- Report generation may timeout for large datasets (async required)
- Deletion can be soft delete (archive) instead of hard delete
- Report access requires user permissions
- Duplicate reports may use cached base data
- Cache invalidation on data changes
- Scheduled report skipped if previous still processing
- Email delivery retry on failure
- File compression for large reports
- Storage cleanup of very old reports
