Business KPI Reporting and Export providing scheduled KPI report generation, multi-format export, KPI comparison reports, and automated distribution enabling leadership to access KPI insights regularly and share with stakeholders.

## Backend API Requirements

### GET /api/business-kpi/reports/saved-reports/

Retrieve list of saved KPI reports.

**Query Parameters:**
- `limit` (optional): Number of reports per page (default: 20)
- `offset` (optional): Pagination offset (default: 0)
- `type` (optional): Filter by report type (executive-summary, financial-ratios, etc.)
- `date_range` (optional): Filter by date range (last-7-days, last-30-days, etc.)
- `search` (optional): Search by report name

**Response Format:**
```
{
  "count": 45,
  "next": "https://api.example.com/business-kpi/reports/saved-reports/?offset=20",
  "previous": null,
  "results": [
    {
      "id": "report-001",
      "name": "May 2026 KPI Summary",
      "type": "executive-summary",
      "generated_at": "2026-05-31T14:30:00Z",
      "generated_by": "User Name",
      "generated_by_id": "user-123",
      "format": "PDF",
      "file_size": 2048576,
      "file_url": "/media/reports/report-001.pdf",
      "status": "completed",
      "parameters": {
        "start_date": "2026-05-01",
        "end_date": "2026-05-31",
        "include_comparison": false
      }
    }
  ]
}
```

## Database Requirements

### Existing Models to Enhance
- **SavedReport Model**: Add KPI-specific fields
  - kpi_report_type (CharField: executive-summary, financial-ratios, etc.)
  - kpi_report_parameters (JSONField: stores report parameters)
  - report_metadata (JSONField: page count, chart count, etc.)
  - expires_at (DateTimeField: auto-delete old reports)

### New Models to Create
- **ScheduledReport**:
  - id (UUID)
  - tenant_id (ForeignKey to Tenant)
  - name (CharField)
  - report_type (CharField)
  - schedule (CharField: cron expression)
  - email_recipients (JSONField: list of email addresses)
  - parameters (JSONField: report parameters)
  - status (CharField: active/inactive/paused)
  - last_run (DateTimeField: when last executed)
  - next_run (DateTimeField: when next scheduled)
  - created_at (DateTimeField)
  - updated_at (DateTimeField)
  - created_by (ForeignKey to User)

- **ReportTemplate**:
  - id (UUID)
  - tenant_id (ForeignKey to Tenant)
  - name (CharField)
  - report_type (CharField)
  - description (TextField)
  - parameters (JSONField: default parameters)
  - created_at (DateTimeField)
  - created_by (ForeignKey to User)
  - last_used (DateTimeField)

- **ReportAccessLog** (optional):
  - id (UUID)
  - report_id (ForeignKey to SavedReport)
  - user_id (ForeignKey to User)
  - action (CharField: download, email, view, delete)
  - accessed_at (DateTimeField)

### Required Indexes
- SavedReport: (tenant_id, created_at DESC), (report_type, created_at)
- ScheduledReport: (tenant_id, status), (next_run)
- ReportTemplate: (tenant_id, report_type)
- ReportAccessLog: (report_id, accessed_at), (user_id, accessed_at)

## Validation & Edge Cases

### Report File Management
- Report file must exist and be accessible
- File size limits enforced
- Cleanup of old reports (>90 days)
- Storage quota per tenant

### Email Validation
- Email recipients must be valid email addresses
- Distribution list validation
- Recipient limit enforcement
- Bounce handling

### Scheduled Reports
- Cron expression must be valid
- Schedule conflicts detected
- Timezone handling for scheduled times
- Execution timeout handling

### Report Generation
- Report type must be valid and available
- Date range validation (start <= end, end <= today)
- Required data must exist for report
- Concurrent generation limits

### Async Job Handling
- Job queue management
- Job timeout handling (>10 minutes)
- Job cancellation support
- Job retry on failure

### Multi-tenant Isolation
- Report access limited to creator's tenant
- Scheduled reports isolated by tenant
- Template access per tenant
- Email recipients limited to tenant users
