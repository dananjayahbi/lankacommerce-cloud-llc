## Backend API Requirements

### GET /api/customer-reports/saved-reports/
Get list of saved reports
- **Query params:** limit, offset, sort_by, filter_type, date_range
- **Response:** [{ id, name, type, generated_at, generated_by, format, file_size, status }]

### GET /api/customer-reports/saved-reports/{id}/
Get report details
- **Response:** { id, name, type, generated_at, file_url, parameters, metadata }

### GET /api/customer-reports/saved-reports/{id}/download/
Download report file
- **Query params:** format (optional, for conversion)
- **Response:** file download

### POST /api/customer-reports/saved-reports/{id}/email/
Email report
- **Request body:** { recipients, subject, message (optional) }
- **Response:** { success: true }

### DELETE /api/customer-reports/saved-reports/{id}/
Delete report
- **Response:** { success: true }

### POST /api/customer-reports/generate/
Generate custom report
- **Request body:** { report_type, start_date, end_date, format, filters: {...} }
- **Response:** { id, status, estimated_time_ms, file_url (when ready) }

### GET /api/customer-reports/{id}/status/
Check generation status
- **Response:** { id, status, progress_percent, error (if failed) }

### GET /api/customer-reports/scheduled-reports/
Get scheduled reports (if enabled)
- **Response:** [{ id, report_type, schedule, next_run, last_run, status }]

### POST /api/customer-reports/scheduled-reports/
Create scheduled report
- **Request body:** { report_type, schedule (cron), email_recipients, parameters }
- **Response:** { id, schedule, next_run }

### PATCH /api/customer-reports/scheduled-reports/{id}/
Update scheduled report

### DELETE /api/customer-reports/scheduled-reports/{id}/
Delete scheduled report
